# 08 — UX/UI Spec (Course + Mock Builder)

## Course layout
Each module page has tabs:
1) Learn
2) Build
3) Validate
4) Simulate
5) Teach-back

## Navigation
- left sidebar: modules with lock/progress
- clear “Next module” CTA after passing teach-back submission

## Builder UI
- Sidebar node palette (searchable)
- Canvas with nodes + edges
- Inspector panel:
  - required fields highlighted
  - inline examples
- Inline warnings:
  - disconnected node
  - missing config

## Validation UI
- show requirements checklist
- show pass/fail per invariant
- each error links to node selection (“Fix this”)

## Simulation UI
- choose test case
- run simulation
- step through logs
- highlight executed nodes
- show contact state before/after

## Teach-back UI
- prompt shown
- rubric checklist visible
- minimum word count
- optional co-pilot “coach my answer”

## Accessibility baseline
- keyboard navigable
- good contrast for errors
- readable typography
